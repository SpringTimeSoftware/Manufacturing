# Planning Screen Specification

## Planning Principles

Planning screens must be real decision workbenches. Static cards, shallow lists, and fallback data are not enough. Planners need live demand, inventory, BOM, route, capacity, purchase, production, and dispatch signals with clear exceptions and conversion actions.

## Screen Specifications

## Planning Override Rules

- All manual overrides require reason, user, timestamp, and downstream impact.
- Planning cannot use missing item UOM, replenishment, lead time, BOM, route, or resource data without explicit exception status.
- Conversion actions must create real PR/PO/WO/job card records or remain disabled.
