# Organization And Resource Setup Screen Specification

## Organization Setup Standard

Organization screens are foundational setup screens. They should not be large blank cards with a few fields. Each screen needs list/search, status filters, a tabbed editor, validation summary, audit trail, and explicit activation rules because downstream production, inventory, planning, quality, and dispatch flows depend on these records.

## Common Audit And Status Behavior

- All setup records require created/updated metadata and an audit trail.
- Activation/deactivation must be explicit and blocked when transactional dependencies exist.
- Changes to calendars, warehouses, bins, machines, and work centers must show downstream impact where possible.
- Setup screens must never expose raw backend terms or implementation messages to business users.
