# Platform And Admin Screen Specification

## Platform Principles

Platform screens are trust screens. They must be compact, production-grade, role-aware, and free of internal implementation copy. PlatformAdmin can see all setup/admin areas, but that access must be an intentional super admin rule or seed, not an accidental bypass.

## Super Admin Rule

The application should support exactly one intentional all-menu demo/admin path: **Super Admin**. This can be implemented as a seeded PlatformAdmin-derived user or a deterministic policy rule that grants all menus and actions to a `SuperAdmin` role. It must still audit actions, respect company/branch context, and never bypass data-validation rules.

## Login Production Requirements

- No implementation copy, framework names, prompt IDs, guarded-demo language, or backend reachability text.
- No huge blank spaces.
- Compact, balanced visual hierarchy.
- Stale refresh token is cleared silently and safely.
- Forgot-password entry remains accessible.
- Authentication errors are clear and production-safe.
- Successful login lands on the role home dashboard with a valid company/branch context.
