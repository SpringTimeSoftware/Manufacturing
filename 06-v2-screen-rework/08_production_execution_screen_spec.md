# Production Execution Screen Specification

## Production Execution Principles

Production execution must preserve the existing backbone while becoming transaction-safe. Web is for supervision, planning, review, and correction. Mobile is for operator action, scan, capture, and proof. Production receipt, scrap, rework, material issue/return, downtime, and shift entry must not be finalized on shallow V1 master-data assumptions.

## Production Stop Gates

- Do not complete production receipt, scrap, rework, or inventory-cost behavior against shallow item/customer/supplier/commercial data.
- Do not allow mobile execution to post irreversible transactions without sync integrity, duplicate protection, and audit.
- Do not expose costing values until costing and landed-cost foundation is approved.
