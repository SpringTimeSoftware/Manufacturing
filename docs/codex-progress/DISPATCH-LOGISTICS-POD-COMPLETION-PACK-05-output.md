## APIs / Services Changed

- `DispatchContracts.cs`
  - Added PCID fields to pack and shipment line DTOs/requests.
  - Added logistics, source-reference, revision snapshot, commercial snapshot, and POD fields to shipment contracts.
  - Added line-level POD confirmation request fields.
- `DispatchService.cs`
  - Shipment creation now validates shipment lines against SO dispatch balance.
  - Shipment creation posts dispatch stock issue through `InventoryPostingService.IssueAsync`.
  - Inventory issue lines persist dispatch/source document references, SO references, bin/lot/serial/PCID dimensions, and revision snapshots.
  - Failed validation occurs inside the dispatch transaction and prevents partial shipment stock posting.
  - POD update now persists receiver/date/contact/remarks/evidence and line delivered/short/damaged quantities.
  - POD cannot be accepted before shipment, and delivered quantity cannot exceed shipped quantity.
  - Shipment creation preserves SO commercial snapshot values and does not recalculate SO pricing/tax/discount.
- `DispatchEntityConfigurations.cs`
  - Configured the new dispatch logistics/POD/source/commercial fields.
