# Architecture V2 Delta Matrix

## Cutover Rule

- Stop the original prompt chain at `P063`.
- Do not resume `P064` until the pre-`P064` remediation sequence in `R001` through `R013` is complete.
- Preserve the current manufacturing execution backbone: BOM, BOQ, MRP, Work Orders, Job Cards, Machine Board, Stage Wise Dashboard, Order Delivery Dashboard, IIS publish-folder deployment, ASP.NET Core, React web, React Native mobile, and SQL Server.

## Legends

- Classification
  - `ADDITIVE`: a new domain can be added beside a usable baseline.
  - `REFACTOR`: the repo already has a related surface, but the model is too shallow and must be reshaped.
  - `DEFERRED`: valid V2 scope, but safe to schedule after foundation work if the pre-`P064` cut line is respected.
- Disposition
  - `KEEP`: preserve as-is.
  - `PATCH`: preserve the baseline and extend it.
  - `REPLACE`: replace the current business shape with a compatibility-led migration path.

## Baseline Area Disposition

| Area | Current baseline | Disposition | Why |
| --- | --- | --- | --- |
| Organization and scope | Company, Branch, Department, Warehouse, Bin, Shift entities and APIs exist. | `KEEP` | The multi-company and branch backbone is usable and should be verified, not rewritten. |
| Measurement foundation | UOM, conversion, measurement profiles, formulas, item UOMs are implemented. | `PATCH` | The measurement layer is strong, but packaging, physical specs, and pricing awareness are missing. |
| Item master | Item, item variant, item UOM, item barcode, generic attachments exist. | `REPLACE` | The current item shape is not ERP-grade and cannot absorb catalog, media, texts, packaging, or customer/vendor references cleanly. |
| Partner master | Customer, customer address, supplier, supplier address, supplier lead time exist. | `REPLACE` | Current customer and supplier models collapse legal, site, contact, consent, credit, and terms responsibilities. |
| Commercial transactions | Quote, Sales Order, Blanket Order, Forecast APIs exist. | `REPLACE` | The transaction shell is useful, but pricing, discount, tax, currency, credit, and dispatch preference foundations are absent. |
| Procurement core | PR, PO, Subcontract Order APIs exist. | `PATCH` | Keep the document skeleton, then add compliance, vendor references, landed cost hooks, and return foundations. |
| Inventory and traceability | Stock balances, stock ledger, reservations, lots, serials, cycle counts exist. | `PATCH` | Preserve the execution ledger, but extend it for packaging, valuation hooks, advanced barcodes, and return lifecycles. |
| Engineering and planning | Routing, BOM, ECO, MPS, MRP, BOQ, capacity patterns exist. | `KEEP` | This is the strongest part of the repo and should be retrofitted to richer masters instead of rebuilt. |
| Manufacturing execution | Work Orders, Job Cards, Downtime, Machine Board wrapper exist. | `KEEP` | This is salvageable and should survive the rebaseline intact. |
| Platform services | Auth, data scope, audit, attachments, notifications, localization exist. | `PATCH` | Reuse the scaffolding, but add consent, item-media semantics, richer templates, and better localization surfaces. |
| SQL asset chain | No ordered migration pack, no SQL DDL pack, no stored-procedure pack beyond one wrapper class. | `REPLACE` | Architecture V2 must restore schema-first and procedure-first discipline before more backend expansion. |
