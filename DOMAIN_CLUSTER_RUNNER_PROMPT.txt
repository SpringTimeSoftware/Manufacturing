GLOBAL NON-NEGOTIABLE GATES
1. No visible touched action may remain dead.
   Every visible touched action must end as:
   - WORKING
   - DISABLED WITH REASON
   - HIDDEN

2. No governed master-linked value may remain unrestricted free text where a source master exists.

3. No numeric/decimal/money/quantity/weight/dimension/rate field may remain a generic unrestricted text field.

4. No active upload/media/document action may remain fake.
   If file/media/doc upload is part of a screen contract, do not merely report “pending”.
   Implement the workflow if it is an internal application workflow that can be completed in this repo.
   Only disable with a clear business-safe reason if the workflow truly depends on an external blocker.
   Use the same core upload/storage/metadata/authorization logic consistently across all screens that use file/media/document upload.

5. In live authenticated mode, do not silently show seeded operational fallback when live API fails.

6. Deep create/edit/detail experiences must use centered modal workspace or full-page workspace.
   No right-drawer deep editors.

7. No internal/scaffold wording in production-facing UI.

8. Do not stop because one screen or one cluster improved.
   Continue until the current cluster is COMPLETE or BLOCKED.
   Continue to the next cluster automatically unless a real blocker appears.

FOR EACH CLUSTER DO THIS EXACT SEQUENCE

PHASE A — Cluster inventory and baseline
- Identify all screen prompt files in this cluster from 001_EXECUTION_MANIFEST.csv.
- Build baseline counts for this cluster only:
  - screens in scope
  - deep editors in scope
  - governed lookup violations
  - numeric field violations
  - dead actions
  - upload/media/document truth violations
  - seeded/live-data truth violations
  - layout/modal violations
  - scroll/overflow violations
  - wording violations

- Update:
  - docs/final-audit/07_screen_issue_register.csv
  - 07-governance/screen_field_violation_matrix.csv
  - 07-ux-governance/action_truth_matrix.csv

PHASE B — Execute every screen prompt in the cluster
For each prompt file in the cluster:
1. Read the prompt file completely.
2. Implement the screen exactly to its completion contract.
3. Do not skip required sections, fields, actions, validations, lookups, screenshots, or output conditions.
4. If a prompt says upload/media/document workflow must exist, implement it if it can be completed in-repo.
5. If an action/workflow truly cannot exist due to an external blocker, disable it with a clear business-safe reason and document the blocker.
6. Do not treat “documented as pending” as completion.
7. Update any shared components or backend support needed to make the screen truthful.

PHASE C — Cluster-wide enforcement pass
After all screen prompts in the cluster are implemented:
- do one sweep across the cluster to remove any remaining:
  - free-text governed fields
  - numeric-as-text fields
  - dead visible actions
  - fake upload/media/doc actions
  - silent seeded live operational data
  - right-drawer deep editors
  - broken scroll/overflow
  - internal wording leaks

PHASE D — Validation
Run:
- npm run typecheck
- npm test
- npm run build
- npm run build:host
- dotnet build src/server/STS.Mfg.sln
- dotnet test src/server/STS.Mfg.sln --no-build
- dotnet publish src/server/STS.Mfg.Host/STS.Mfg.Host.csproj -c Release

If validation fails:
- attempt to auto-fix trivial validation blockers first
- rerun validation
- only stop if a real blocker remains

PHASE E — Screenshot evidence
Capture screenshots for all touched primary screens and modals in the cluster.
If a screen scrolls, max 3 screenshots:
- top
- middle
- bottom
Do not loop.

Store under:
- docs/codex-review-screens/MASTER-COMPLETION-CLUSTER-RUNNER-01/<cluster-name>/

PHASE F — Cluster summary and gate
Create:
- docs/codex-progress/<cluster-name>-output.md

The cluster summary must include:
- screens touched
- screens fully compliant
- screens still partial
- lookup violations fixed
- numeric field violations fixed
- dead actions removed/disabled/wired
- upload/media/document truth issues fixed
- seeded/live-data issues fixed
- layout/scroll issues fixed
- remaining blockers in the cluster
- validation results
- screenshot folder path

The cluster is COMPLETE only if:
- critical violations for the cluster = 0
- no dead visible actions remain in the cluster
- touched governed field violations = 0
- touched numeric field violations = 0

If the cluster is BLOCKED:
- report exact blockers
- then stop the overall run

If the cluster is COMPLETE:
- continue automatically to the next cluster
